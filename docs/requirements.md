# 极客雷达 · 产品需求文档

> 版本: 1.0 · 日期: 2026-06-03 · 作者: 鱼皮

---

## 一、背景与目标

鱼皮作为 AI 编程博主，需要**第一时间**感知 AI 大模型领域的热点动态（模型发布、工具更新、重大研究等），但不能依赖人工手动搜索。

**目标：** 构建一个轻量级 AI 驱动的热点监控工具，自动发现热点、验证真实性、实时推送通知。

---

## 二、核心功能

### F1 · 关键词监控

| 项 | 描述 |
|---|---|
| 功能 | 用户手动输入要监控的关键词（如 "GPT-5 发布"、"Claude 4"、"Cursor 更新"） |
| 触发 | 当抓取到相关内容时，使用 **AI（OpenRouter）** 验证是否为真实相关内容（排除标题党/无关引用） |
| 通知 | 验证通过（置信度 ≥ 65%）后，立即发送**浏览器通知 + 邮件通知** |
| 防误报 | AI 二次验证机制：先进行关键词文本预筛，再调用 AI 判断语义相关性 |

### F2 · 热点自动发现

| 项 | 描述 |
|---|---|
| 功能 | 每隔一段时间（默认 30 分钟）自动从多个数据源抓取内容 |
| 处理 | AI 对每条内容打**热度分（1-10）**，生成一句话中文摘要，归类（模型发布/工具更新/研究/融资/讨论） |
| 展示 | 用户通过 Web 页面的**雷达视图**浏览所有热点 |
| 手动刷新 | 用户可点击按钮立即触发一次抓取 |

### F3 · 通知中心

- 浏览器系统通知（Notification API，需用户授权）
- 邮件通知（SMTP，可在设置页配置）
- 应用内实时推送（SSE Server-Sent Events）
- 未读告警角标计数

### F4 · 配置中心

- OpenRouter API Key 配置
- 邮件 SMTP 参数配置
- 检查间隔时间配置（默认 30 分钟）
- AI 模型选择

## 四、产品形态

- **Web 应用**（Phase 1）：响应式网页，独特雷达视觉设计
- **Agent Skills**（Phase 2）：封装为 Claude Code Skills，支持 AI 调用

---

## 五、非功能需求

- **轻量**：SQLite 本地数据库，无需额外服务
- **自部署**：`.env` 配置，`npm run dev` 即可运行
- **唯一性**：UI 与同类工具有明显差异（雷达视觉风格）
- **容错**：单个数据源失败不影响整体运行；AI 服务不可用时降级为文本匹配

## 六、验收标准

- [ ] 添加关键词后，当相关新闻出现时，浏览器收到通知
- [ ] 邮件通知可正常发送（配置 SMTP 后）
- [ ] 热点列表每 30 分钟自动刷新
- [ ] 雷达动画流畅，热点展示清晰
- [ ] 手动刷新按钮有效
- [ ] 设置页可保存 API Key 和邮件配置
